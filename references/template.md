# 文档模板与大纲示例

本文件给出 `knowledge-summary` skill 在 Step 2（输出大纲）和 Step 3（填充正文）使用的统一骨架。所有总结文档都按这个结构来写，方便读者扫读。

---

## 一、Step 2 大纲格式（先发给用户确认）

输出形式建议如下，便于用户增删：

```markdown
# {主题} 学习总结 · 大纲

## 一、概述
- 1.1 是什么 / 解决了什么问题
- 1.2 适用场景

## 二、核心概念
- 2.1 概念 A
- 2.2 概念 B
- 2.3（前置）xxx

## 三、工作原理
- 3.1 数据流 / 生命周期
- 3.2 关键算法 / 数据结构

## 四、源码剖析
- 4.1 入口函数：`<repo>/<path>#Lxx`
- 4.2 关键实现：`<repo>/<path>#Lxx`
- 4.3 调用链：HTTP handler → router → policy → ...

## 五、为什么这样设计
- 5.1 设计动机
- 5.2 与方案 X 的对比

## 六、实战使用
- 6.1 环境准备 / 安装命令
- 6.2 端到端最小示例（输入 / 命令 / 输出）
- 6.3 调参 / 排错

## 七、进阶与边界
- 7.1 性能注意事项
- 7.2 已知坑点 / 最佳实践

## 八、周边知识地图（按需）
- 相关知识点 + 一句话简介 + 官方链接

## 九、面试高频考点与陷阱（必备，≥ 12 道题）
- 9.1 基础原理题（≥ 2）
- 9.2 陷阱 / 易错题（≥ 2）
- 9.3 大局 / 系统设计题（≥ 2，必须含"规模放大后的问题"）
- 9.4 对比 / 取舍题（≥ 2）
- 9.5 源码 / 实现题（≥ 2）
- 9.6 演进 / 趋势题（≥ 2）

## 十、参考资料
- 官方文档
- 官方源码
- RFC / 规范（如有）

## 计划填入的源码 / 示例清单
- 源码片段：`<repo>/<path>#Lxx`（讲什么）
- 端到端示例：`<命令 / 配置>`（演示什么）
- Mermaid 图：调用链 / 架构图
```

> 用户确认后再进入 Step 3 写正文。

## 二、Step 3 正文骨架

## 三、使用提示

- 章节顺序固定，但**子节数量按主题复杂度增减**；
- 所有 `<...>` 占位符在写正文时都必须替换为真实内容；
- 不要把"模板里写过的占位结构"原样保留进最终文档；
- 每段源码都遵守 `SKILL.md` 三、代码处理规则：单次展示 + 中文注释直接写在代码里。
